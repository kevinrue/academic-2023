Set up GitHub Actions

Overview

This section introduces a workflow to set up continuous integration for a working R package.

Open a new git branch

Note: an earlier section will explain how to protect the master branch.

At this point, the master branch of your GitHub repository should be protected. As always, we will work on a branch and merge to the master branch when the continuous integration feature is ready.

git checkout -b ci

Load usethis

library(usethis)

Create a README file

Note: this should be moved to an earlier section, as it is not part of the continuous integration setup per se.

If you haven’t already, use use_readme_md() to create a README.md.

use_readme_md()

Set up the template ‘R CMD check’ action

use_github_action_check_full()

As per documentation:

This action installs the last 5 minor R versions and runs R CMD check via the R BiocStyle::CRANpkg("rcmdcheck") package on the three major OSs (linux, macOS and Windows).

Upon completion, use_github_action_check_full() generates Markdown content that displays a badge indicating the status of the GitHub Action. This content is both copied to your clipboard, and printed in the console. It should look like:

  <!-- badges: start -->
  [![R build status](https://github.com/user/repo/workflows/R-CMD-check/badge.svg)](https://github.com/user/repo/actions)
  <!-- badges: end -->

Paste this content in the badges: section of your README.md file.

Edit the GitHub Action configuration

First, remove the restriction on branches for the push event. This will trigger the action on every push to any branch.

on:
  push:
  pull_request:
    branches:
      - master

Next, keep only the latest ubuntu version, mainly to reduce the number of configurations run as separate jobs. For instance, this would look something like the following (update to the relevant version):

    strategy:
      fail-fast: false
      matrix:
        config:
          - {os: macOS-latest,   r: 'devel'}
          - {os: macOS-latest,   r: '4.0'}
          - {os: windows-latest, r: '4.0'}
          - {os: ubuntu-16.04,   r: '4.0', rspm: "https://packagemanager.rstudio.com/cran/__linux__/xenial/latest"}

When working with a Bioconductor package, BiocManager should be installed during the Query dependencies step. This allows BiocManager::repositories() to be called to add Bioconductor package repositories to be added to the global session options. In turn, this allows remotes::dev_package_deps() to query all repositories when identifying dependencies, as it uses the default argument value repos = getOption("repos").

- name: Query dependencies
  run: |
    install.packages(c('remotes', 'BiocManager'))
    options(repos = c(getOption("repos"), BiocManager::repositories()))
    saveRDS(remotes::dev_package_deps(dependencies = TRUE), "depends.Rds", version = 2)
  shell: Rscript {0}

Again, when working with a Bioconductor package, we add Bioconductor package repositories to the global session options during the Install dependencies step. Similarly, the remotes::install_deps(dependencies = TRUE) uses the default argument value repos = getOption("repos").

In this step, you may also add any dependency that is not explicitly declared in the DESCRIPTION file of your package, or installed as an indirect dependency, as well as any esoteric development version of your dependencies.

- name: Install dependencies
  run: |
    options(repos = c(getOption("repos"), BiocManager::repositories()))
    remotes::install_deps(dependencies = TRUE)
    remotes::install_cran("rcmdcheck")
  shell: Rscript {0}
